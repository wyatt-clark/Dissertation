\section{Introduction}
%%%%%%%%%%%%%%%%%%%%%%
%%%%%%%%%%%%%%%%%%%%%%

Although there are notable exceptions \citep{Dunker2001a,Dunker2001}, the structure and function of a protein  are indelibly related; a protein's ability to induce chemical or physical change can be characterized, at some level, by  illustrating what structural characteristics (or lack thereof) endow that protein with its particular abilities.  While there are well established methods for experimentally resolving the structure of a protein (NMR, x-ray crystallography), these methods are costly, tedious, time consuming and rarely guarantee success.  Because of these difficulties the rate of growth of the compendium of known protein structures is far outpaced by the growth of known sequences; growth largely driven by high-throughput sequencing efforts.  In light of the important relationship between structure and function computational methods can potentially bridge the gap between the set of all known coding sequences and their associated protein structures.

I believe we might be misleading readers to think we are implying the opposite when we are actually making a more subtle point: the physical confirmation of a protein dictates its propensity to behave in certain ways.

The same is true when considering the function of a protein.  Experimentally characterizing a protein's function is expensive and time consuming.  Furthermore, with many organisms certain experiments, such as knockout studies in humans, are also unethical to conduct.  \emph{In-silico} methods also hold the greatest promise for functionally characterizing the large number of unannotated protein sequences that have been discovered.

Perhaps the simplest, and most intuitive manner in which structure and function can be inferred is through homology, or its common proxy sequence similarity. Traditionally, protein sequences are represented as a string of letters representing amino acids.  Alignment programs like BLAST \citep{Altschul1990} do not explicitly consider the physiochemical properties associated with each amino acid, although it could be argued that substitution matrices like PAM \citep{Dayhoff1978} and BLOSUM \citep{Henikoff1992} implicitly encode the physiochemical similarities between amino acids.  

In this paper we have made a conscious choice to represent proteins, not as a sequence of letters or 3-D coordinates, but as a set of physiochemical and structural characteristics associated with each amino acid.  Although kernel methods can potentially be used to compare objects without explicitly representing them in feature space, we used a kernel function to encode each variable length property vector; which we refer to as a time series sequence, as a fixed length vector suitable for use in a supervised learning framework.  Kernel have been widely used in computational biology \citep{schoolkopf2004kernel}, finding applications in the identification of functional residues \citep{Vacic2010,Xin2011}, the prediction of protein-protein interactions \citep{BenHur2005}, the inference of function \citep{Vert2002,DeBie2007}, and the categorization of secondary strucutre \citep{Leslie2002}.  Our kernel function is novel in the way that we handle real valued, as opposed to discrete data, utilizing $k$-means clustering to discretize a continuous feature space.

While it may be beneficial to represent an amino acid as a set of real values based on physiochemical and structural properties; as opposed to a characters that don't explicitly encode any biological or chemical information, the shift from a representation that takes on an enumerable number of values to one that is uncountably infinite poses problems.  For example, one common kernel representation of an object is the simple count kernel (detailed in \Cref{sec:propertykernels}).  This representation has been used in the classification of text documents, protein sequences, as well as [insert tons of examples here].  A count kernel counts the number of times a single character, $n$-mer, or word appears in a sequence.  Each sequence, or document, is then represented as a fixed length vector, whose length is defined by the cardinality of the set of possible $n$-length words that can be created by the relevant alphabet.  For example, a count kernel using $n=3$ length ``mers" to represent a  protein would result in each sequence being represented as a $20^{3}$ length vector.  Representing each amino acid, as a real value poses a problem because each amino acid can be assigned an infinitely uncountable set of values, resulting in a theoretically infinite feature vector when encoded using a count kernel.  

We solve this problem in a novel way by discretizing $n$ length vectors using k-means clustering (in this case we use $m$ means).  Each $n$ length vector sampled from the larger series of data can then be discretized and represented by the centroid from the clustered data it is closest to.  We chose to use $k$-means clustering as opposed to simply creating a lattice in $n$ dimensional space, as done by \citet{Tuytelaars2007}, because sampled property vectors do not fill the space evenly, but instead cluster around evolutionarily conserved or sterically preferred ``spaces".

Finally, we compare the efficacy of explicitly using sequence properties to a sequence similarity based spectral kernel \citep{Leslie2002} in the tasks of predicting protein function and SCOP fold.
